Windows Error Reporting

 

WER의 중요성

응용 프로그램의 호환성 오류를 저장하는 Shimcache와 유사하게, Windows Error Reporting은 악성 프로그램의 침입, 침해 등 공격에 대한 조사 방법으로 사용될 수 있습니다. 매우 정교하게 만들어진 악성 프로그램이 아닌 이상. 공격 대상 시스템의 버전에 완벽하게 호환하도록 개발 할 수 없기에 Windows Error Reporting에서 그 흔적을 추적할 수 있을 것입니다.

 다만, 이는 사용자가 임의로 설정 / 해제를 수행할 수 있기에 분석 대상의 설정 정보를 미리 확인할 필요가 있음을 유의해야 합니다.

 

WER의 경로

%UserProfile%\AppData\Local\Microsoft\Windows\WER

SystemDrive%\ProgramData\Microsoft\Windows\WER

 

WER의 구성

 

기본적으로 C:\Users\{사용자}\ProgramData\Microsoft\Windows\WER 경로에는 ReportArchive, ReportQueue, Temp 폴더가 존재합니다.

Windows Error Reporting 파일에서는, 문제가 발생한 파일과 발생 시각을 확인할 수 있습니다. 발생 시각은 Decode 과정이 필요하며, Windows FileTime 형식으로 저장되어 있습니다.

이 외에도 문제가 발생한 파일의 경로와, Loaded된 Module 목록, App ID, 발생한 Event의 이름 등을 추가적으로 확인할 수 있습니다.

 

gpedit(그룹 정책 편집기)에서 관리 템플릿 - Windows 오류 보고 폴더에서 해당 기능을 사용 or 사용 안함 상태로 변경할 수 있습니다.
